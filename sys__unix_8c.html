void Sys_DoStartProcess ( char *  cmdline)

Actually forks and starts a process.

Parameters
[in]cmdlineExecution string
Note
UGLY HACK:
Sys_StartProcess works with a command line only
If this command line is actually a binary with command line parameters, the only way to do this on unix is to use a system() call but system doesn't replace the current process, which leads to a situation like:
wolf.x86–spawned_process.x86
in the case of auto-update for instance, this will cause write access denied on wolf.x86:
wolf-beta/2002-March/000079.html
we hack the implementation here, if there are no space in the command line, assume it's a straight process and use execl otherwise, use system...
The clean solution would be Sys_StartProcess and Sys_StartProcess_Args..
