16 INTRODUCTION
17 ============
18 
19 Enemy Territory: Legacy is based on the [raedwulf-et](https://bitbucket.org/tcmreastwood/raedwulf-et/)
20 project which in turn is based on the [GPL'd source code](https://github.com/id-Software/Enemy-Territory) of Wolfenstein: Enemy Territory.
21 
22 The main goals of the project are fixing bugs, cleaning up the codebase and adding useful features
23 while remaining compatible with the ET 2.60b version.
24 
25 The Legacy mod is the default mod shipped with ET: Legacy. It aims to add many useful features and
26 improvements, while staying close to the original gameplay, as well as being lightweight and extensible
27 through Lua scripts.
28 
29 For more information consult our [changelog](https://dev.etlegacy.com/projects/etlegacy/wiki/Changelog).
30 
31 ET: Legacy development is a collaborative effort done in an open, transparent and friendly manner.
32 Anyone is welcome to join our efforts!

53 ### Game data
54 
55 Wolfenstein: Enemy Territory is a free release, and can be downloaded from [Splash Damage](https://www.splashdamage.com/games/wolfenstein-enemy-territory/).
56 
57 This source release contains only the engine and mod code but not any game data,
58 which is still covered by the original EULA and must be obeyed as usual.
59 
60 In order to run ET: Legacy you will need to copy the original assets files
61 (*pak0.pk3*, *pak1.pk3* and *pak2.pk3*) to the etmain folder.
62 
63 
64 ### Compatibility with Enemy Territory 2.60b
65 
66 ET: Legacy remains compatible with the ET 2.60b version as much as possible.
67 
68 Please note that ET: Legacy is *not* compatible with PunkBuster enabled servers.
69 ET: Legacy clients also cannot connect to servers running the ETPro mod.
70 
71 
72 ### Linux 64 bit
73 
74 Please remember that 64 bit ET: Legacy clients can only connect to servers running
75 mods providing a 64 bit version. You will be able to play 32 bit-only mods only if
76 you compile ET: Legacy on a 32 bit system or cross-compile it for 32 bit architecture
77 on a 64 bit system.
78 
79 At the moment, only the Legacy mod is available in 64 bit version, while all other
80 existing mods are available in 32 bit only version.
81 
82 In case you are a running a 64 bit system, you probably might want to use the
83 **bundled libraries** which are located in a separate *etlegacy-libs* repository and
84 can be automatically downloaded using the `git submodule` command. See the next
85 section for more details.

88 DEPENDENCIES
89 ============
90 
91 * **CMake** (compile-time only)
92 * **OpenGL**
93 * **GLEW**
94 * **SDL**
95 * **ZLib**
96 * **MiniZip**
97 * **libjpeg-turbo** or **libjpeg**
98 * **libcurl** (optional, enabled by default)
99 * **Lua** (optional, enabled by default)
100 * **Ogg Vorbis** (optional, enabled by default)
101 * **Theora** (optional, enabled by default)
102 * **Freetype** (optional, enabled by default)
103 * **SQLite** (optional, enabled by default)
104 * **OpenAL** (optional, enabled by default)
105 * **Jansson** (optional)
106 
107 
108 Grab info about current lib versions from our wiki page [ET: Legacy Libs_Changelog](https://dev.etlegacy.com/projects/etlegacy/wiki/Libs_Changelog)
109 
110 To get the latest source code install [git](http://git-scm.com/) and
111 clone our repository hosted at [Github.com](https://github.com/etlegacy/etlegacy):
112 
113  $ git clone git://github.com/etlegacy/etlegacy.git
114 
115 If the required dependencies are not installed on your system run:
116 
117  $ git submodule init
118  $ git submodule update
119 
120 This downloads the essential dependencies into the `libs/`directory. You can choose
121 whether to use bundled libraries instead of the system ones by changing the
122 `BUNDLED_LIBS` variable in the CMakeList.txt configuration file. You can then select
123 which bundled libraries to use by toggling the respective `BUNDLED_XXX` variable.

126 COMPILE AND INSTALL
127 ===================
128 
129 To install the binaries system-wide, you need to compile ET: Legacy with hardcoded
130 fs_basepath.
131 
132 The following variables can be adjusted in CMake:
133 
134  * **INSTALL_DEFAULT_BASEDIR**: sets default *fs_basepath*, i.e. where etl and etlded
135  executables look for data files. In most cases it is CMAKE_INSTALL_PREFIX+INSTALL_DEFAULT_MODDIR.
136  Defaults to empty value, because we want *fs_basepath* to be the current working directory
137  when not installing the game system-wide.
138 
139  * (optional) **INSTALL_DEFAULT_BINDIR**: Location for executables. Appended to CMAKE_INSTALL_PREFIX.
140  Defaults to "bin".
141 
142  * (optional) **INSTALL_DEFAULT_MODDIR**: Location for libraries and paks. Appended to
143  CMAKE_INSTALL_PREFIX. Defaults to "share/etlegacy" and then "legacy" is appended to it.

146 ### Linux
147 
148 Install required dependencies.
149 
150 * option A: **easybuild**
151 
152 In terminal, run:
153 
154  $ ./easybuild.sh
155 
156 ET: Legacy will be installed in `~/etlegacy`.
157 
158 * option B: **command line**
159 
160 In terminal, run:
161 
162  $ mkdir build && cd build && cmake ..
163 
164 To compile, run:
165 
166  $ make
167 
168 If you wish to install ET: Legacy system-wide, run:
169 
170  # make install
171 
172 Be sure to set the CMake variables (see above) beforehand.
173 
174 
175 **NOTES:**
176 
177  * Even if you have a 64 bit linux distribution which provides 32 bit versions of all
178  the required libraries, you might also need the development libraries (-devel packages)
179  installed on your system.
180 
181  * In order to compile the jpeg-turbo library properly you will need the **nasm** assembler.

184 ### Crosscompiling on Linux with MinGW-w64
185 
186 In terminal, run:
187 
188  $ mkdir build && cd build
189  $ cmake -DCMAKE_TOOLCHAIN_FILE=../cmake/Toolchain-cross-mingw-linux.cmake ..
190  $ make
191 
192 By default, MinGW name is set to *i686-w64-mingw32*. You may have to change it in
193 `cmake/Toolchain-cross-mingw-linux.cmake` depending on how it is called on your system.

196 ### Windows
197 
198 Install:
199 
200  1. [Visual Studio Community](http://www.visualstudio.com/) with the _Desktop Development with C++_ workload
201  2. [CMake](http://www.cmake.org/) and make sure it is added to your system PATH
202 
203 * option A: **easybuild**
204 
205  1. run easybuild.bat
206 
207 ET: Legacy will be installed in `My Documents\ETLegacy-Build`.
208 
209 * option B: **Visual Studio**
210 
211  1. create a `build` directory inside the directory which contains ET: Legacy sources
212  2. open *Visual Studio Command Prompt* in the Start menu
213  3. change directory with `cd` to the newly created build directory
214 
215 In the command prompt, run:
216 
217  cmake -G "NMake Makefiles" -DBUNDLED_LIBS=YES .. && nmake
218 
219 or
220 
221  cmake -G "Visual Studio 16" -DBUNDLED_LIBS=YES ..
222 
223 and open the resulting project in Visual Studio.
224 
225 
226 **NOTES:**
227 
228  * If compilation of bundled libraries is aborted for any reason, you will probably need to clean the
229  libs directory and start over. This can be done by executing `git clean -df && git reset --hard HEAD`
230  inside `libs/` directory.
231 
232  * If the build fails during libcurl compilation because of missing *sed* utility, download it from
233  [GnuWin](http://gnuwin32.sourceforge.net/packages/sed.htm) and place it into your system path or
234  copy it into `MSVC/VC/bin`. It also comes with Git and can be placed into your system path
235  automatically if you select that option during Git installation.

238 ### Mac OS X
239 
240 Install:
241 
242  1. [Xcode](https://developer.apple.com/xcode/downloads/)
243  2. [Homebrew](http://brew.sh/)
244  3. [Homebrew Cask](http://caskroom.io/)
245 
246 Then brew the following packages in the terminal app:
247 
248  $ brew cask install xquartz
249  $ brew install --universal gnu-sed cmake glew sdl2 minizip jpeg-turbo curl lua libogg libvorbis theora freetype sqlite openal-soft
250 
251 The --universal flag ensures both 32bit and 64bit libraries are installed. Although your system curl library supports both architectures, you also need to install its headers.
252 
253 * option A: **easybuild**
254 
255 In Terminal, run:
256 
257  $ ./easybuild.sh
258 
259 This will put an 'etlegacy' folder into your user folder.
260 
261 * option B: **command line**
262 
263 In terminal, run:
264 
265  $ mkdir build && cd build && cmake ..
266 
267 To compile, run:
268 
269  $ make
270 
271 If you wish to install ET: Legacy system-wide, run:
272 
273  # make install
274 
275 Be sure to set the CMake variables (see above) beforehand.
